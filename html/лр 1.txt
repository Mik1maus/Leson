Лабораторна робота №1
Контрольні запитання та відповіді

1. Назвіть основні елементи HTML-документа. Якими тегами визначається його структура?
Відповідь: Основні елементи: <!DOCTYPE html>, <html>, <head>, <body>. Структура задається тегами <html> (кореневий елемент), <head> (службова інформація) і <body> (вміст сторінки).

2. Як закоментувати фрагмент HTML-коду?
Відповідь: За допомогою конструкції:
<!-- коментар -->

3. Як у мові HTML позначаються спеціальні сутності (такі, як символи <, & чи >)?
Відповідь: За допомогою HTML-сутностей (entity):
< → &lt;
> → &gt;
& → &amp;

4. Призначення елемента <!DOCTYPE>.
Відповідь: <!DOCTYPE html> повідомляє браузеру, що документ написаний у стандарті HTML5 і допомагає коректно його відображати.

5. Теги секції заголовка.
Відповідь: У секції <head> найчастіше використовують теги: <title>, <meta>, <link>, <style>, <script>.

6. Призначення метатегів.
Відповідь: Теги <meta> містять службову інформацію про сторінку (кодування, опис, ключові слова, налаштування для пошукових роботів, адаптивність тощо).

7. Як, використовуючи метатеги, заборонити індексацію сторінки пошуковими роботами?
Відповідь: Додати в <head>:
<meta name="robots" content="noindex, nofollow">

8. Атрибути тега link.
Відповідь: Основні атрибути тега <link>: rel (тип зв’язку), href (адреса файлу), type (тип вмісту), sizes (для іконок), media (для стилів під певні пристрої).

9. Як задати сайту Favicon?
Відповідь: У секцію <head> додати:
<link rel="icon" href="favicon.ico">

10. Скільки рівнів заголовків має HTML?
Відповідь: Шість рівнів заголовків: від <h1> до <h6>.

11. Чи можна задавати користувацькі атрибути у HTML?
Відповідь: Так, можна задавати атрибути, що починаються з data-, наприклад: <div data-id="123">.

12. У чому різниця між тегами <br/> та <BR>?
Відповідь: За дією різниці немає: обидва створюють перенесення рядка. <br> — сучасний запис у нижньому регістрі, <BR> — застарілий варіант у верхньому регістрі.

13. Наведіть приклади тегів для форматування тексту (курсив, жирний і т.д.).
Відповідь: Жирний текст: <b> або <strong>; курсив: <i> або <em>; підкреслення: <u>; надрядковий індекс: <sup>; підрядковий індекс: <sub>.

14. У чому різниця між елементами <div> і <p>?
Відповідь: <p> призначений для абзаців тексту. <div> — універсальний блочний контейнер для групування будь-яких елементів. Семантично <p> означає параграф, а <div> — просто блок.

15. У чому різниця між блоковими та рядковими елементами?
Відповідь: Блокові елементи (block) займають всю ширину рядка і починаються з нового рядка (<div>, <p>, <h1>, <table>). Рядкові елементи (inline) займають тільки потрібну ширину і не переносять рядок (<span>, <a>, <b>, <i>, <img>).

16. Що таке рядково-блокові елементи? Наведіть приклади.
Відповідь: Це елементи, які поводяться як рядкові, але можуть мати ширину й висоту, подібно до блокових. Приклади: <img>, <input>, <button>, <select>, <textarea>.

17. Які теги використовуються для семантичної верстки?
Відповідь: Для семантичної верстки використовують: <header>, <footer>, <nav>, <main>, <section>, <article>, <aside>, <figure>, <figcaption>.

18. Наведіть приклад створення маркованих, нумерованих списків та списків визначень.
Відповідь:
Маркований список (ul):
<ul>
  <li>Пункт 1</li>
  <li>Пункт 2</li>
</ul>

Нумерований список (ol):
<ol>
  <li>Перший</li>
  <li>Другий</li>
</ol>

Список визначень (dl):
<dl>
  <dt>Термін</dt>
  <dd>Пояснення</dd>
</dl>

19. Як змінити вигляд нумерації списку?
Відповідь: Для <ol> використовують атрибут type, наприклад:
<ol type="A"> (букви), <ol type="a">, <ol type="I"> (римські цифри), <ol type="1"> (звичайні числа).

20. Які теги використовуються для створення таблиць?
Відповідь: Основні теги таблиць: <table> — таблиця, <tr> — рядок, <td> — комірка, <th> — комірка-заголовок, <caption> — підпис таблиці.

21. Як об'єднати комірки таблиці по горизонталі?
Відповідь: Використати атрибут colspan у тегу <td> або <th>:
<td colspan="2">Текст</td>

22. Наведіть приклад абсолютного та відносного гіперпосилань.
Відповідь:
Абсолютне:
<a href="https://google.com">Google</a>

Відносне (в межах сайту):
<a href="about.html">Про нас</a>

23. Які атрибути тега a є обов'язковими (без яких валідацію не буде пройдено)?
Відповідь: Обов’язковий атрибут — href (адреса посилання).

24. Як здійснити перехід до певних розділів інших документів? Як перейти в межах поточного документа?
Відповідь: Для переходу до розділу використовують ідентифікатор id.
У межах поточного документа:
<a href="#top">Вгору</a>
<h2 id="top">Початок</h2>

До розділу іншого документа:
<a href="about.html#rules">Перейти до розділу</a>

25. Як вказати відкриття гіперпосилання у новій вкладці?
Відповідь: Додати атрибут target="_blank":
<a href="page.html" target="_blank">Відкрити</a>

26. Як за допомогою мови HTML можна створити гіперпосилання, яке при натисканні замість переходу пропонує надіслати E-Mail, використовуючи поштовий клієнт користувача?
Відповідь: Використати схему mailto:
<a href="mailto:example@gmail.com">Написати листа</a>

27. Які види комп'ютерної графіки ви знаєте?
Відповідь: Основні види: растрова графіка та векторна графіка.

28. Чим відрізняється векторна графіка від растрової? У яких випадках доцільно застосовувати растрову графіку, а яких векторну?
Відповідь: Растрова графіка складається з пікселів, добре підходить для фотографій, але втрачає якість при збільшенні. Векторна графіка описується формулами (лінії, криві), добре масштабується без втрати якості, підходить для логотипів, схем, іконок.

29. Формати графічних файлів для векторної та растрової графіки.
Відповідь: Векторні формати: SVG, EPS, AI. Растрові формати: JPEG, PNG, GIF, WebP, AVIF, BMP.

30. Які особливості графічних форматів WebP, AVIF, GIF, JPEG, PNG?
Відповідь:
JPEG — стиснення з втратою якості, добре для фото.
PNG — без втрати якості, підтримує прозорість.
GIF — підтримує прості анімації, обмежена кількість кольорів.
WebP — сучасний формат, менший розмір файлу, підтримка прозорості та анімацій.
AVIF — дуже ефективне стиснення, ще менший розмір, ніж WebP, сучасний формат.

31. Які існують способи додати на сторінку SVG-зображення?
Відповідь: Основні способи:
1) Як файл через <img>:
<img src="image.svg" alt="">
2) Вставити код SVG прямо в HTML:
<svg>...</svg>
3) Використати <object> або <iframe> з посиланням на SVG.

32. Які атрибути тега img є обов'язковими (без яких валідація не буде пройдена)?
Відповідь: Обов’язкові атрибути: src (шлях до зображення) та alt (текстовий опис зображення).

33. Що визначає атрибут decoding для елемента <img>?
Відповідь: Атрибут decoding підказує браузеру, коли декодувати зображення:
"async" — асинхронно, "sync" — одразу, "auto" — браузер сам обирає спосіб.

34. Для чого використовуються теги map та area?
Відповідь: Для створення карти зображення, коли різні ділянки одного зображення є окремими посиланнями.
<map> описує карту, <area> — окремі області (круг, прямокутник тощо).

35. У чому полягає схема data:URI для додавання зображень на сторінку?
Відповідь: Зображення кодується в Base64 і вбудовується прямо в атрибут src, наприклад:
<img src="data:image/png;base64,....">
Так файл зображення не зберігається окремо, а міститься всередині HTML.

36. Для чого використовується тег form? Наведіть приклади керуючих елементів.
Відповідь: <form> використовується для створення форми і надсилання даних користувача на сервер або іншу сторінку.
Приклади керуючих елементів: <input>, <textarea>, <select>, <option>, <button>, <label>.

37. Відмінності використання елементів <input type="button"> и <button> для створення кнопки.
Відповідь: <input type="button"> створює кнопку без вмісту всередині, текст задається атрибутом value.
<button> може містити будь-який HTML-вміст (текст, іконки, теги), наприклад: <button><b>OK</b></button>.

38. Що таке фрейми? Який тег використовується для створення фрейму?
Відповідь: Фрейм — це область сторінки, де відображається інший документ.
У сучасному HTML для вбудовування сторінок використовується тег <iframe src="..."></iframe>. Старі теги <frame> і <frameset> вважаються застарілими.

39. Як ініціалізувати локальний репозиторій у СКВ Git?
Відповідь: У терміналі в папці проекту виконати команду:
git init

40. Як дізнатися про поточне становище локального репозиторію?
Відповідь: Використати команду:
git status

41. Для чого використовується команда git add?
Відповідь: Команда git add додає файли до індексу (staging area) перед створенням коміту. Наприклад:
git add index.html

42. Як відправити версовані зміни на віддалений репозиторій?
Відповідь: Спочатку зробити коміт, потім надіслати його на віддалений репозиторій:
git commit -m "Повідомлення коміту"
git push
